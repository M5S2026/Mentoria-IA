### Notas para Implementacao

1. **Tom geral:** Premium mas humano. Sem exclamacoes excessivas. Sem capslock. Sem emojis na LP (emojis permitidos apenas nas mensagens do concierge/mentores para mobile).

2. **Mobile-first:** Todo texto deve ser legivel em telas pequenas. Paragrafos curtos (max 3-4 linhas em mobile). Headlines de no maximo 2 linhas em mobile.

3. **Hierarquia visual:** Usar tamanho de fonte e peso para guiar o olho. Headlines > Subheadlines > Corpo. Nao depender de cor sozinha para hierarquia.

4. **CTAs:** Botoes com texto acionavel e especifico. Evitar "Saiba Mais" generico. Cada CTA deve dizer exatamente o que acontece ao clicar.

5. **Depoimentos:** Substituir por reais assim que disponiveis. Para lancamento, usar beta testers com disclaimer ou remover secao temporariamente.

6. **Numeros da social proof bar:** Atualizar conforme plataforma cresce. Para lancamento, usar numeros de beta/pre-lancamento.

7. **Urgencia:** So usar escassez real (turma limitada a X vagas, data de inicio definida). Nunca usar timer falso ou escassez artificial.
