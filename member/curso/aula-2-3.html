Dashboard/ Modulo 2/ Aula 3

Graficos, dados e recursos avancados

Elevando o nivel do seu app

Nosso app "Minhas Financas" ja funciona, mas vamos adicionar recursos que fazem um app parecer profissional: graficos interativos, exportacao de dados e mais.

Adicionando graficos com Chart.js

Chart.js e a biblioteca mais popular para graficos em paginas web. O Claude Code sabe usa-la perfeitamente. Envie este prompt:

Adicione graficos ao app Minhas Financas usando Chart.js (inclua
via CDN). Crie uma secao de "Analise" abaixo da lista com:

1. Grafico de PIZZA: distribuicao de despesas por categoria
   - Cores diferentes para cada categoria
   - Mostrar porcentagem ao passar o mouse

2. Grafico de BARRAS: receitas vs despesas dos ultimos 6 meses
   - Barras verdes para receitas, vermelhas para despesas
   - Eixo X com nomes dos meses

3. Grafico de LINHA: evolucao do saldo ao longo do tempo
   - Linha verde quando positivo, vermelha quando negativo

Os graficos devem atualizar automaticamente quando uma transacao
e adicionada ou removida.

Exportar dados para CSV

Adicione um botao "Exportar CSV" que gera um arquivo CSV com
todas as transacoes. O CSV deve ter colunas: Data, Tipo,
Categoria, Descricao, Valor. O arquivo deve baixar automaticamente
quando o usuario clicar.

Exportar relatorio em PDF

Adicione um botao "Gerar Relatorio PDF" que cria um PDF com:
- Titulo: "Relatorio Financeiro - [mes/ano]"
- Resumo com totais
- Tabela de transacoes do mes
- Grafico de pizza com distribuicao de gastos
Use a biblioteca html2pdf.js via CDN.

Modo claro/escuro

Adicione um toggle no header para alternar entre modo escuro e
modo claro. Salve a preferencia no localStorage. O modo claro
deve ter fundo branco (#f5f5f5), cards com sombra suave e texto
escuro. Transicao suave entre os modos.
Sobre bibliotecas externas:

Quando voce menciona bibliotecas como Chart.js ou html2pdf.js, o Claude Code automaticamente inclui via CDN (link externo). Voce nao precisa instalar nada. Apenas mencione o nome da biblioteca e ele cuida do resto.

Validacao de formulario

Melhore a validacao do formulario:
- Valor nao pode ser zero ou negativo
- Descricao deve ter pelo menos 3 caracteres
- Mostrar mensagens de erro em vermelho abaixo de cada campo
- Destacar campos invalidos com borda vermelha
- Limpar erros quando o usuario comeca a digitar

Design responsivo aprimorado

Melhore o layout mobile:
- Graficos empilhados verticalmente no mobile
- Formulario com campos em coluna unica
- Lista de transacoes com swipe para deletar (opcional)
- Resumo com cards 2x2 no mobile ao inves de 4 em linha
- Fonte um pouco maior no mobile para melhor leitura

Resultado esperado

Ao final desta aula, seu app deve ter:

  • 3 graficos interativos que atualizam em tempo real
  • Exportacao para CSV e PDF
  • Modo claro/escuro
  • Validacao robusta de formulario
  • Design responsivo profissional

Isso e um app completo que voce poderia mostrar em um portfolio ou ate adaptar para um cliente!
