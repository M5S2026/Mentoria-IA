Planejamento do projeto

Antes de construir, planeje

O maior erro de quem comeca a usar IA para criar apps e pular direto para o desenvolvimento sem planejar. Mesmo com o Claude Code fazendo o trabalho pesado, voce precisa saber o que quer construir antes de pedir para ele construir.

O Template de Brief do Projeto

Use este template sempre que comecar um novo projeto:

=== BRIEF DO PROJETO ===

Nome do Projeto: [nome simples e descritivo]
Objetivo: [o que este app resolve?]
Publico-alvo: [quem vai usar?]

Funcionalidades essenciais (MVP):
1. [funcionalidade principal]
2. [funcionalidade 2]
3. [funcionalidade 3]

Funcionalidades extras (v2):
1. [nice-to-have 1]
2. [nice-to-have 2]

Design:
- Estilo: [moderno, minimalista, colorido, corporativo]
- Cores: [cor principal, cor de destaque]
- Referencia: [site ou app que voce gosta o visual]

Dados:
- [que tipo de dados o app manipula?]
- [de onde vem os dados? usuario digita? importa CSV?]

Plataforma: [web, mobile, desktop]

Exemplo real: Rastreador de Financas Pessoais

Vamos planejar o app que vamos construir nas proximas aulas:

=== BRIEF DO PROJETO ===

Nome: Minhas Financas
Objetivo: Ajudar usuarios a controlar receitas e despesas mensais
Publico-alvo: Qualquer pessoa que quer organizar suas financas

Funcionalidades essenciais (MVP):
1. Adicionar receitas e despesas com categoria, valor e data
2. Ver resumo mensal (total de receitas, despesas, saldo)
3. Grafico de pizza mostrando gastos por categoria

Funcionalidades extras (v2):
1. Exportar dados para CSV
2. Grafico de evolucao mensal (linha)
3. Modo escuro/claro

Design:
- Estilo: Moderno, limpo, dark mode
- Cores: fundo #0a0a1a, destaque verde #00ff88 para receitas,
  vermelho #e94560 para despesas
- Referencia: apps de banco como Nubank

Dados:
- Usuario digita manualmente
- Salvos no localStorage do navegador

Plataforma: Web (responsivo para mobile)
Dica de ouro:

Comece SEMPRE pelo MVP (Minimum Viable Product). Crie a versao mais simples que funciona primeiro, teste, e so depois adicione funcionalidades extras. Isso evita projetos abandonados e garante que voce tenha algo funcional rapidamente.

Quebrando o projeto em etapas

Com o brief pronto, divida o desenvolvimento em passos pequenos:

  1. Estrutura basica: HTML com o layout principal (header, formulario, lista, resumo)
  2. Estilo visual: CSS com o tema escuro e cores definidas
  3. Funcionalidade de adicionar: Formulario que adiciona transacoes na lista
  4. Calculo do resumo: Somar receitas, despesas e calcular saldo
  5. Persistencia: Salvar dados no localStorage
  6. Graficos: Adicionar grafico de pizza com Chart.js

O prompt inicial

Com o planejamento pronto, voce pode enviar o brief inteiro como prompt para o Claude Code. Na proxima aula, vamos fazer exatamente isso e construir o app passo a passo.

Exercicio

Preencha o template de Brief para um projeto que voce gostaria de criar. Pode ser qualquer coisa: uma calculadora, um organizador de tarefas, um portfolio, um quiz. Tenha o brief pronto para a proxima aula - vamos usa-lo!
